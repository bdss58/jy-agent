## 2026-04-25 14:45 [ship]
Shipped web_search cascade upgrade (commits b1a614e, b4617bb on main, +11 ahead of origin)

Background: user asked me to evaluate web_search (DDG-only HTML scrape) and research how popular AI agents do search. Key external findings:
- Claude Code WebSearch returns title+URL only (drops encrypted_content), Anthropic's server-side tool ~Brave-powered, $10/1k searches (mikhail.io reverse-eng).
- ChatGPT/o3-deep-research grounds on Bing Search.
- Cursor/Cline use Tavily or SerpAPI.
- OSS agents (OpenClaw, Open WebUI) standardize on SearxNG self-hosted.

Decisions:
1. Surgical Chrome-tier removal: deleted _CHROME_EXTRACT_SEARCH_JS + _STRATEGY_MAP_SEARCH from web_fetch. Kept _fetch_chrome as last-resort tier-4 for JS-heavy non-search pages (Twitter/Reddit/Zhihu/Weibo/Bilibili). Chrome SERP scraping was slow (3-10s vs 300ms), serialized on single browser, defeated by Google bot detection.
2. New web_search cascade: SearxNG (env-gated) → DDG → Brave → Mojeek. First engine returning ≥3 results wins. WEB_SEARCH_ENGINE env forces single engine.
3. Verified all selectors against live HTML 2026-04-25:
   - Brave: a.l1 is the stable anchor (svelte-<hash> classes rotate, ignore them)
   - Mojeek: ul.results-standard > li > h2 a.title + p.s for snippet
4. Brave HTML rate-limits aggressively (HTTP 429) from cloud IPs and intermittently from residential — cascade absorbs this cleanly.

Tests: 13 new unit tests + 1 live integration. All 468 tests in suite pass.

Operational note for user: to enable SearxNG, `docker run -d -p 8888:8080 searxng/searxng` then `export SEARXNG_URL=http://localhost:8888`. DDG remains zero-config default.
